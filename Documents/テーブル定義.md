# ImaginAI - テーブル定義

## history（履歴エントリ）

保存方式: ファイルベース（履歴保存ディレクトリ配下にエントリごとに1ディレクトリ）

| カラム名 | 型 | 説明 |
|---|---|---|
| id | string (UUID) | 主キー - 一意識別子 |
| createdAt | string (ISO 8601) | エントリ作成日時 |
| updatedAt | string (ISO 8601) | 最終更新日時 |
| model | string | 生成に使用したAPIモデルID |
| modelDisplayName | string | UI表示用のモデル名 |
| prompt | string | 生成プロンプトテキスト |
| negativePrompt | string | ネガティブプロンプトテキスト（任意） |
| aspectRatio | string | アスペクト比（例: "1:1", "16:9"） |
| quality | string | 出力解像度（"1k", "2k", "4k"） |
| numberOfImages | number | 生成枚数（1-4） |
| referenceImagePaths | string[] | 使用した参照画像のフルパス配列 |
| generatedImagePaths | string[] | 生成画像ファイルのフルパス配列（1エントリ1画像） |
| imageWidth | number? | 生成画像の幅（ピクセル） |
| imageHeight | number? | 生成画像の高さ（ピクセル） |
| fileSize | number? | 生成画像のファイルサイズ（バイト） |

### ストレージ構造

```text
<historyDir>/
  images/
    <uuid>.png             # 生成画像ファイル（PNG固定）
    <uuid>.json            # HistoryEntry メタデータ
  thumbnails/
    <uuid>.jpg             # サムネイル画像（JPEG、長辺最大300px）
```

- 1画像 = 1 UUID。複数枚生成時はUUIDも複数。
- ファイル名はUUIDで統一（png, json, jpg で拡張子のみ異なる）。

## settings（アプリケーション設定）

保存方式: JSONファイル（`<userData>/settings.json`）

| カラム名 | 型 | 説明 |
|---|---|---|
| language | string | UI言語（"ja" または "en"） |
| theme | string | テーマ設定（"light", "dark", "system"） |
| historyDir | string | 履歴保存ディレクトリのフルパス |

## api-keys（APIキー）

保存方式: 暗号化ファイル（`<userData>/api-keys.enc`、Electron safeStorage使用）

| カラム名 | 型 | 説明 |
|---|---|---|
| (プロバイダキー) | string | プロバイダ名をキーとしたAPIキー値（例: "gemini"） |
