# ImaginAI - システム仕様

## 1. システム概要

ImaginAIは、AI画像生成APIを専用GUIで操作するデスクトップアプリケーションである。Electronで構築され、初期段階ではGoogle Gemini APIに対応し、将来的に他のAPIプロバイダへの拡張が可能な設計となっている。

## 2. アーキテクチャ

### 2.1 プロセスモデル

- **Mainプロセス**: IPC通信、ファイルI/O、API呼び出し、設定管理、APIキー暗号化、ウィンドウ管理を担当する。
- **Preloadプロセス**: `contextBridge`を通じてMainプロセスとRendererプロセスを安全に橋渡しする。
- **Rendererプロセス**: React + MUI によるUIを担当する。

### 2.2 ディレクトリ構造

```
src/

    index.ts              # アプリエントリ、ウィンドウ生成、コアIPC
    ipc/index.ts          # アプリケーションIPCハンドラ

      api-key-service.ts  # safeStorageによるAPIキー暗号化

      gemini-service.ts   # Gemini API通信
    utils/
      console-bridge.ts   # DevToolsコンソール転送
  preload/

  renderer/

    stores/
```

## 3. 機能仕様

### 3.1 画像生成

- ユーザーはモデルを選択し、パラメータ（アスペクト比、解像度、生成枚数、出力フォーマット、セーフティフィルター）を設定し、プロンプトを入力し、任意で参照画像を添付する。
- パラメータはモデルの対応状況に基づいて検証される（例: 21:9は対応モデルのみ、4Kは対応モデルのみ）。
- 参照画像はファイル選択、ドラッグ＆ドロップ、または履歴エントリから追加できる。サムネイルとして表示され、個別に削除可能。
- 「生成」ボタンでAPIリクエストを送信する。生成中はボタンが無効化され、ローディングインジケータが表示される。
- 成功時は結果が自動的に履歴に保存される。失敗時はエラーメッセージが表示される。
- 履歴件数が設定された上限（デフォルト: 200件）を超えている場合、生成がブロックされる。

### 3.2 履歴管理

- 各生成結果はメタデータ（JSON）と画像ファイルを含むディレクトリとして保存される。
- 履歴はサムネイルグリッドでフレックスレイアウト表示され、更新日降順でソートされる。
- 左クリックでモードレスビューアウィンドウに画像を表示する。
- 右クリックでコンテキストメニューを表示: プロンプトに追加、名前をつけて保存、パラメータ復元、削除。
- ヘッダーメニュー: すべて削除、すべて圧縮保存して削除（ZIPエクスポート）。
- 履歴パネルの高さは上辺のドラッグで調整可能。
- プロンプトテキストによる部分一致検索でフィルタリング可能。

### 3.3 画像ビューア

- モードレスウィンドウとして開く（複数同時に開くことが可能）。
- 初期サイズは画像の等倍ピクセルサイズ。画面サイズの90%を超える場合は縮小して表示する。
- ウィンドウリサイズ時に画像は縦横比を維持して拡大・縮小する。

### 3.4 設定

- **言語**: i18nextによる日本語/英語切り替え。変更は即時反映。
- **テーマ**: ライト/ダーク/システム（OS追従）。変更は即時反映。
- **APIキー**: Electronの`safeStorage`で暗号化保存。マスク表示と表示/非表示トグル付き。接続テストボタンでAPIキーの有効性を検証可能。
- **履歴保存先**: ユーザーが任意のディレクトリを選択可能。変更時に既存履歴の移動有無を確認するダイアログを表示。

## 4. データフロー

### 4.1 生成フロー

1. Renderer: ユーザーがパラメータを設定し、生成ボタンをクリック
2. Renderer -> Main (IPC): `generation:execute`でGenerationParamsを送信
3. Main: 履歴件数が上限を超えていないか検証
4. Main: Gemini API呼び出し（Imagen predictまたはGemini generateContent）
5. Main: 生成画像とメタデータを履歴ディレクトリに保存
6. Main: Electron nativeImageでサムネイルを生成
7. Main -> Renderer (IPC): HistoryEntryを返却
8. Renderer: 履歴一覧を再読み込み

### 4.2 API通信

- **Imagenモデル**（例: imagen-3.0）: `/v1beta/models/{model}:predict`エンドポイントを使用
- **Geminiモデル**（例: gemini-2.0-flash）: `/v1beta/models/{model}:generateContent`エンドポイントを使用し、`responseModalities: ['TEXT', 'IMAGE']`を指定
- 参照画像はリクエスト時にローカルファイルパスからBase64エンコードして送信
- APIキーはリクエスト時に暗号化ストレージから取得

## 5. セキュリティ

- APIキーはElectronの`safeStorage`により暗号化して保存される。
- RendererプロセスはAPIキーに直接アクセスせず、すべてのAPI操作はMainプロセス経由のIPC通信で処理される。
- コンテキスト分離とPreloadスクリプトにより、WebコンテンツとNode.js APIの間に安全な境界を確保する。

## 6. 拡張性

- モデル定義は`shared/constants.ts`に一元管理されている。新モデル追加時はこのファイルのみを更新する。
- 各モデル定義には対応パラメータが含まれ、UIの有効/無効制御が自動的に行われる。
- モデル定義のAPIプロバイダフィールドにより、将来的な複数プロバイダ対応に備えている。
- APIキーストレージはプロバイダ別にキー管理されており、複数APIキーに対応可能。

## 7. 定義定数

| 定数名 | 値 | 説明 |
|---|---|---|
| HISTORY_MAX_COUNT | 200 | 生成がブロックされる履歴上限件数 |
| GENERATION_COUNT_MIN | 1 | 1回の生成における最小枚数 |
| GENERATION_COUNT_MAX | 4 | 1回の生成における最大枚数 |
| DEFAULT_ASPECT_RATIO | "1:1" | デフォルトのアスペクト比 |
| DEFAULT_QUALITY | "1k" | デフォルトの出力解像度 |
| DEFAULT_OUTPUT_MIME_TYPE | "image/png" | デフォルトの出力フォーマット |
| DEFAULT_SAFETY_FILTER_LEVEL | "block_some" | デフォルトのセーフティフィルター |
| DEFAULT_NUMBER_OF_IMAGES | 1 | デフォルトの生成枚数 |
| THUMBNAIL_SIZE | 200 | サムネイルのサイズ（ピクセル） |
